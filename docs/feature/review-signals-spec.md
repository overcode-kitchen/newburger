# 후기 신호 3종 + 규칙 집계 스펙 (MVP 확장)

## 1) 목표

- 후기의 신뢰 신호를 텍스트/별점 외에 구조화된 값으로 축적합니다.
- 사용자 입력 부담을 줄이기 위해 3개 항목 모두 선택 입력(optional)으로 둡니다.
- 입력값이 없는 과거/신규 텍스트 후기는 키워드 규칙 집계로 보조 추론합니다.

## 2) 입력 항목 정의 (선택)

- `taste_vote`
  - `tasty`: 맛있다
  - `so_so`: 애매하다
  - `bad`: 맛없다
- `value_vote`
  - `good`: 좋다
  - `normal`: 보통
  - `poor`: 아쉽다
- `repurchase_vote`
  - `yes`: 있다
  - `maybe`: 고민
  - `no`: 없다

## 3) DB 반영 원칙

- `public.reviews`에 nullable 컬럼으로 추가합니다.
- 기존 후기 레코드는 `null` 유지(백필 필수 아님).
- RLS insert 정책은 별점/코멘트 검증을 유지하고, 신규 컬럼은 enum 허용값만 통과하도록 확장합니다.

## 4) 집계 규칙

- 공식 집계(우선): 구조화 컬럼 응답값 기반
- 보조 집계(보완): 구조화 값이 없는 후기의 `comment`를 키워드 규칙으로 판정
- 최종 표시 기본값:
  - 라벨: 최다 득표 항목
  - 비율: `해당 라벨 건수 / 항목 응답 총건수`
  - 표본: `n=응답 총건수`
- 최소 표본 수:
  - `n < 5`면 "데이터 수집 중" 표시

## 5) UI 표기 원칙

- 상세 페이지 요약 카드 예시:
  - 맛 만족도: 맛있다 61% (n=42)
  - 가성비: 좋다 48% (n=31)
  - 재구매 의사: 있다 55% (n=29)
- 규칙 집계가 섞이면 출처를 분리 표기:
  - 직접 선택 기반
  - 텍스트 추론 기반(실험적)

## 6) 처리 우선순위

1. 사용자가 직접 선택한 값 사용
2. 해당 값이 없고 텍스트가 있으면 규칙 추론 사용
3. 둘 다 없으면 미집계

## 7) 제외/예외 처리

- 텍스트 길이 2자 미만은 추론 제외
- 긍/부정 동률이면 미판정
- 노이즈 문구(광고/링크 위주)는 추론 제외 후보로 관리

## 8) 롤아웃 순서

1. DB 컬럼 추가 + 타입 반영
2. 후기 폼에 선택 입력 3종 추가(모두 optional)
3. 상세 요약 카드 UI 추가
4. 키워드 규칙 집계 연결
5. 운영 중 키워드 사전 보정
